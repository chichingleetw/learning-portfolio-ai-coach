# 對話示範

本文件整理自原始 PDF《對話示範-學習歷程自述教練》，以一位想申請資訊工程系的學生為例，展示學習歷程自述 AI 教練如何依照五階段流程互動。

## 怎麼開始

打開 Antigravity 後，直接用自然語言說明需求即可：

```text
我想申請資工系，幫我準備備審資料
```

```text
幫我寫學習歷程自述
```

```text
我參加過科展但不知道怎麼寫
```

教練會自動進入引導流程。它不會直接寫好文章，而是透過提問協助學生把經驗具體化。

## 階段 1：盤點經驗

學生：

```text
我想申請資工系，幫我寫備審資料。
```

教練會先說明不會代寫，接著請學生列出高中三年印象深刻的活動、課程或比賽。

學生可能一開始只說：

```text
我參加過資訊社，高二做過一個專題，然後校隊打過排球。
```

教練會一次聚焦一個經驗，用 STAR 框架追問：

- 這個專題在做什麼？
- 你負責哪個部分？
- 過程中遇到什麼困難？
- 最後怎麼解決？
- 這件事對你有什麼影響？

如果學生回答：

```text
我們做一個記帳 App，我寫前端介面那部分。其實就學到很多啦。
```

教練會偵測到「學到很多」太空泛，改追問：

```text
你寫前端的時候，遇到最卡的一個問題是什麼？最後怎麼解決的？
```

學生補充：

```text
按鈕點下去資料一直存不進去，我查了兩三天才發現是非同步的問題，後來學會用 async/await 才解決。
```

這時素材就從「學到很多」變成可被看見的具體證據：

- 卡關兩三天。
- 自己查出非同步問題。
- 學會使用 `async/await` 解決資料寫入問題。

### 階段 1 產出示例

| 編號 | 經驗名稱 | 時間 | 角色 | 關鍵事件 | 收穫與影響 |
|------|----------|------|------|----------|------------|
| 1 | 記帳 App 專題 | 高二 | 前端開發 | 卡在非同步資料寫入，自學 `async/await` 解決 | 學會 debug 流程與查文件 |
| 2 | 資訊社 | 高一到高二 | 教學長 | 帶學弟妹從零學 Python | 練習把複雜概念講簡單 |
| 3 | 排球校隊 | 高一到高三 | 隊員 | 連續兩季練習、區賽第四 | 培養規律與抗壓 |

教練也會做補漏檢查。例如素材幾乎都是技術類時，會提醒學生思考是否還有人際、社團、挫折或合作經驗。

## 階段 2：回顧歷程

當素材足夠後，教練會引導學生退一步看全貌。

可能的追問：

```text
你覺得這幾件事之間，有什麼共通的地方嗎？
```

學生若回答：

```text
好像都是我自己想辦法解決問題。
```

教練會接著引導學生確認脈絡，而不是直接替學生定義人生主題。

這個階段的重點是讓學生自己發現：

- 哪些經驗反覆出現同一種能力。
- 哪個時刻讓方向變得更明確。
- 現在的自己和高一相比有什麼不同。

## 階段 3：整理亮點

以資訊工程系為例，常見審查重點包含：

- 邏輯與問題拆解。
- 程式解題經驗。
- 自主學習能力。

教練會請學生思考：

```text
自學 async/await 解決問題的經驗，你覺得展現了什麼能力？
```

接著整理成能力亮點對照表。

| 校系看重的能力 | 對應經驗 | 具體證據 |
|----------------|----------|----------|
| 自主學習與 debug 毅力 | 記帳 App 專題 | 卡關兩三天後，自學 `async/await` 解決非同步資料寫入問題 |
| 邏輯與表達 | 資訊社教學 | 把 Python 概念拆解後教給學弟妹 |
| 規律與抗壓 | 排球校隊 | 連續兩季規律訓練並參與區賽 |

能力標籤必須有具體事實支撐，不能只寫「我很有毅力」「我很會合作」。

## 階段 4：連結未來

教練會把過去經驗連到未來學習計畫。

若學生只說：

```text
我想學 AI，因為覺得很酷。
```

教練會追問：

```text
是什麼具體事情讓你對 AI 產生興趣？是看到某個應用、某篇報導，還是自己做過什麼？
```

學生可能補充：

```text
做記帳 App 的時候，我想如果它能自動分類消費就好了，後來查了才知道那要用機器學習。
```

這時就能整理出「過去到未來」的因果鏈：

```text
做記帳 App
→ 想讓消費自動分類
→ 查到機器學習
→ 想進資工系深入學 AI
→ 未來想做能實際幫助使用者的智慧應用
```

這比「我對 AI 充滿熱情」更具體，也更能看見動機來源。

## 階段 5：表達修整

素材和脈絡整理完成後，教練會請學生先自己寫初稿。

建議結構可能是：

1. 開場：用記帳 App 卡關的具體畫面切入。
2. 學習歷程：寫專題與資訊社兩個核心經驗。
3. 反思與能力：說明自學、debug 與表達能力。
4. 未來展望：從記帳 App 連到想學 AI 的原因。

若初稿出現：

```text
我從小就對電腦充滿熱情，相信自己能在資訊領域發光發熱。
```

教練不會直接重寫，而是提出修改方向：

1. 「我從小就對電腦充滿熱情」太像套話，建議改用記帳 App 卡關的真實畫面開場。
2. 「發光發熱」太空泛，建議刪掉，改說自己具體想學什麼或解決什麼問題。
3. 被動語氣可以改成主動語氣，讓文章更像學生自己的聲音。

最後，教練會提醒學生將完成稿交由指導教師審閱。

## 其他校系的提醒

### 醫藥衛生類

若學生說：

```text
我在醫院當過志工，應該寫我很有愛心吧。
```

教練會提醒「有愛心」太像口號，並追問具體場景：

```text
當志工時，有沒有哪個具體時刻，讓你對照顧別人有不一樣的體會？
```

### 社會心理類

若學生說：

```text
這段志工經歷讓我收穫滿滿，深受感動，對未來方向更清楚了。
```

教練會指出這句有多個空泛詞，並請學生聚焦一個具體收穫：

```text
你說的收穫，最大的那一個是什麼？可以描述當下發生了什麼嗎？
```

## 總結

1. 直接用白話說明想申請的校系與做過的事。
2. 被追問細節是正常的，因為具體證據比抽象形容更重要。
3. 文章必須由學生自己寫，教練負責問對問題、整理方向與協助修改。
4. 最終版本建議交由指導教師審閱。
