# 安裝與使用說明

本文件整理自原始 PDF《Antigravity-學習歷程教練-安裝與使用說明》，改寫為適合 GitHub 閱讀的版本。

## 1. 安裝 Antigravity

Antigravity 2.0 是 Google 推出的 AI 代理人工作平台，可以在電腦上執行代理人並處理指定專案中的任務。本專案使用標準版 Antigravity，不需要安裝工程師用的 IDE 版本。

### 下載與安裝

1. 使用瀏覽器開啟官方下載頁：<https://antigravity.google/download>
2. 選擇符合電腦作業系統的安裝檔。
3. 執行安裝檔，依照畫面指示完成安裝。
4. 啟動 Antigravity，使用 Google 帳號登入並完成驗證。
5. 安全與資料使用政策畫面可依指示點選 Next。
6. 外掛選擇畫面若只是一般教學用途，可以先略過並完成設定。

### 小提醒

- 登入需要 Google 帳號。若學校或公司帳號有限制，可以改用個人 Gmail。
- 安裝完成後，可以從桌面或應用程式列表開啟 Antigravity。

## 2. 建立專案資料夾

Antigravity 使用「專案」來組織工作。一個專案可以包含一個或多個資料夾，代理人只會讀取你授權給它的資料夾。

建議先建立一個專案資料夾，例如：

```text
學習歷程教練/
```

常見放置位置：

```text
Windows: C:\Users\你的名字\Documents\學習歷程教練
macOS: /Users/你的名字/Documents/學習歷程教練
```

在 Antigravity 中建立專案：

1. 點選右側面板的 `Select Project`。
2. 選擇 `New Project`。
3. 點選 `Add Folder`，選取剛建立的「學習歷程教練」資料夾。
4. 權限設定先使用預設值即可。
5. 完成後進入此專案。

## 3. 放入 AGENTS.md

`AGENTS.md` 是本專案的核心規則檔。你可以把它想成這個 Antigravity 專案的使用守則。

一般聊天框中的指令只對當次對話有效；寫進 `AGENTS.md` 的規則則會在每次新 session 開始前被讀取，並在流程中持續套用。

本專案的 `AGENTS.md` 會要求代理人：

- 扮演學習歷程自述教練。
- 依照五階段流程引導學生。
- 不代寫、不編造、不誇大。
- 對空泛內容追問具體例子。
- 最終提醒學生交由指導教師審閱。

### 放置方式

將 `AGENTS.md` 放在專案資料夾的最外層：

```text
學習歷程教練/
└── AGENTS.md
```

注意事項：

- 檔名必須是 `AGENTS.md`。
- 不要改成 `agents.txt`、`規則.md` 或其他名稱。
- 建議放在專案根目錄，不要放進子資料夾。
- 修改 `AGENTS.md` 後，建議開啟新的對話，讓 Antigravity 重新讀取規則。

## 4. 確認是否生效

放好檔案後，在 Antigravity 聊天框輸入：

```text
你現在的角色是什麼？要遵守哪些規則？
```

如果代理人回答自己是學習歷程自述教練，並提到五階段流程、不代寫、不編造等原則，就表示設定已生效。

## 5. 開始使用

不需要背特殊指令，直接用自然語言描述需求即可。

常見開場：

```text
我想申請資工系，幫我準備備審資料
```

```text
幫我寫學習歷程自述
```

```text
我參加過科展，但不知道怎麼寫
```

教練不會直接生成一篇可提交的文章，而是會一步步追問細節，協助學生把模糊經驗整理成具體素材。

## 6. 使用流程

| 階段 | 學生要做的事 | 會得到的產出 |
|------|--------------|--------------|
| 1. 盤點經驗 | 列出課程、社團、競賽、志工、專題等經驗，回答追問 | 經驗素材清單 |
| 2. 回顧歷程 | 思考經驗間的共通點與轉折 | 成長脈絡 |
| 3. 整理亮點 | 比對目標校系看重的能力 | 能力亮點對照表 |
| 4. 連結未來 | 說明就讀動機、想學課程與未來方向 | 過去到未來的敘事 |
| 5. 表達修整 | 自己寫初稿，再依修改方向調整 | 自述文章修改建議 |

## 7. 教練的底線

- 不代寫整段文章，只給結構、提問和修改方向。
- 不編造經驗，所有內容都必須來自學生說過的話。
- 不誇大普通經驗，保持真實比例。
- 不提供可直接照抄的範文。
- 完成後提醒交由指導教師審閱。
